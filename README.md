## 1. 引言：KeyForge是什么？

### 1.1 一句话理解

**KeyForge 是一个能让您在电脑后台模拟键盘按键的工具**，就像有一个隐形的手在帮您按键盘，而您自己可以继续做其他事情。

### 1.2 用生活例子理解

想象一下这样的场景：
- 您正在玩一个游戏，需要反复按 `F1` 键来放技能
- 您正在测试一个软件，需要重复输入同样的内容
- 您打开了5个聊天窗口，需要给每个窗口发送相同的消息

**没有KeyForge时**：您需要自己反复按键，手会酸，还会出错。  
**有了KeyForge后**：设置好按键，它就会自动帮您按，想按多少次就按多少次，想按多久就按多久。

### 1.3 为什么叫"KeyForge"？

"Key"是按键，"Forge"是锻造。这个名字的含义是：**为您锻造出最强大的按键工具**。无论您面对什么样的程序、什么样的游戏，KeyForge都能找到发送按键的方法。

### 1.4 谁需要KeyForge？

| 人群 | 用途 |
|------|------|
| **游戏玩家** | 测试连招、自动挂机、多开同步操作 |
| **软件测试工程师** | 自动化测试、重复性操作、压力测试 |
| **系统管理员** | 批量配置、远程操作、脚本辅助 |
| **普通用户** | 解决按键无效问题、演示教学 |
| **开发者** | 学习Windows消息机制、调试程序 |

### 2.2 第二步：找到目标窗口

2. 观察您的"记事本"，会看到出现一个"**A**"
3. 查看日志区域，会显示发送成功的记录

## 3. 核心功能详解

### 3.1 窗口枚举设置

窗口枚举是KeyForge的"眼睛"，决定了它能"看"到哪些窗口。

#### 3.1.1 按窗口标题

**这是什么？** 搜索窗口标题栏中包含您输入关键词的窗口。

**举例：**
- 输入"**记事本**" → 会找到所有标题包含"记事本"的窗口
- 输入"**征途**" → 会找到"征途2"、"征途怀旧版"等游戏窗口

**适用场景：** 窗口标题固定或包含明显特征

#### 3.1.2 按程序名

**这是什么？** 搜索进程名（.exe文件名）或加载的模块名（.dll/.dat文件）包含关键词的窗口。

**重点说明：** 这是KeyForge的强大功能！很多游戏主程序名是`client.exe`，但会加载`zhengtu.dat`模块。输入`zhengtu.dat`就能找到游戏窗口，即使标题变化也不怕。

**举例：**
- 输入"**notepad.exe**" → 找到记事本程序
- 输入"**zhengtu.dat**" → 找到征途游戏窗口
- 输入"**chrome**" → 找到所有Chrome浏览器窗口

**适用场景：** 窗口标题会变化，但程序固定的情况（常见于游戏）

#### 3.1.3 两者兼顾

**这是什么？** 同时满足标题和程序名两个条件才匹配。

**举例：** 开了两个"记事本"，一个编辑"文件A.txt"，一个编辑"文件B.txt"。输入标题"文件A"和程序名"notepad.exe"，就能精确控制第一个记事本。

#### 3.1.4 调试模式

**这是什么？** 开启后，点击刷新时会显示所有可见窗口的详细信息（标题、类名、PID、进程名、模块）。

**什么时候用？** 当您找不到目标窗口时，开启调试模式，在日志中查看所有窗口，找到目标窗口的特征关键词。

### 3.2 按键设置

#### 3.2.1 按键类型

KeyForge支持超过100种按键，包括：

| 类别 | 按键示例 |
|------|----------|
| 字母 | A-Z |
| 数字 | 0-9 |
| 功能键 | F1-F24 |
| 方向键 | Up, Down, Left, Right |
| 控制键 | Space, Enter, Esc, Tab, Backspace, Delete, Insert, Home, End, PageUp, PageDown |
| 修饰键 | Ctrl, Alt, Shift, Win |
| 小键盘 | Num0-Num9, NumMult, NumPlus, NumMinus, NumDiv, NumDot |
| 多媒体键 | VolumeUp, VolumeDown, VolumeMute, MediaNext, MediaPrev, MediaPlayPause |
| 浏览器键 | BrowserBack, BrowserForward, BrowserRefresh, BrowserSearch, BrowserHome |
| 符号键 | `, -, =, [, ], \, ;, ', ,, ., / |
| 其他 | CapsLock, PrintScreen, ScrollLock, Pause, Menu, Sleep |

**高级用法：** 可以直接输入虚拟键码，如 `0x41` 代表A，`0x20` 代表空格。这可以发送预设列表中没有的按键。

#### 3.2.2 组合键

**这是什么？** 同时按下多个键，如 `Ctrl+C`、`Alt+F4`。

**如何使用：** 勾选组合键复选框（Ctrl、Alt、Shift、Win），然后选择主按键。

**举例：**
- `Ctrl` + `C` → 复制
- `Ctrl` + `Alt` + `Del` → 打开安全选项
- `Alt` + `F4` → 关闭窗口
- `Win` + `R` → 打开运行对话框

**注意：** Fn键是硬件级按键，无法通过软件模拟。

#### 3.2.3 发送间隔

每两次按键发送之间的等待时间。范围：0.01秒 ~ 10秒。

- **间隔太小**：目标程序可能反应不过来
- **间隔太大**：效率太低
- **建议**：游戏场景0.3-0.5秒，软件场景0.1-0.3秒

#### 3.2.4 重复次数

- **>0的数字**：发送指定次数后自动停止
- **0**：**无限循环**！持续发送直到手动点击停止

**无限循环的应用：**
- 挂机刷怪：设置好按键，让游戏角色自动打怪
- 压力测试：持续向软件发送大量按键
- 演示展示：反复演示某个操作

### 3.3 60种后台按键方式详解

这是KeyForge最核心、最强大的功能！不同的程序对键盘消息的处理方式不同，KeyForge提供了60种"武器"，总有一种能攻破对方的"防线"。

#### 3.3.1 基础消息方式（0-9）

| 序号 | 名称 | 通俗解释 | 什么时候用 |

| 6 | WM_CHAR | "发送最终的字符" | 需要输入文本时 |
| 7 | WM_IME_CHAR | "输入法专用字符" | 输入中文时 |
| 8 | WM_SYSKEYDOWN | "系统快捷键" | 发送Alt组合键 |
| 9 | PostMessage完整参数 | "带详细地址的快递" | 结合2和5的优点 |

#### 3.3.7 如何选择发送方式？

**简单原则（按顺序尝试）：**

1. **第1种 (SendMessage)** → 首选，稳定可靠
2. **第2种 (PostMessage)** → 需要快速发送时
3. **第3-4种 (keybd_event/SendInput)** → 后台无效时尝试
4. **第5-9种 (带参数的WM消息)** → 程序校验消息参数时
5. **第20-21种 (硬件/DirectInput)** → 游戏专用
6. **第22-24种 (驱动/DLL注入)** → 终极手段，需管理员权限

## 4. 实战场景举例

### 4.1 游戏测试场景

#### 场景1：征途游戏后台挂机

**需求：** 在《征途》游戏中，自动按`F1`、`F2`、`F3`释放技能，无限循环。

**设置步骤：**
1. **枚举窗口**：选择"按程序名"，输入"**zhengtu.dat**"，刷新并全选
2. **按键设置**：
   - 按键类型：`F1`
   - 重复次数：`0`（无限循环）
3. **开始发送**：点击开始，程序会持续按F1
4. **动态切换**：在发送过程中，将按键类型改为`F2`，程序立即开始按F2

#### 场景2：多开同步操作

**需求：** 开了3个游戏窗口，想同时按`W`键前进。

**设置步骤：**
1. **枚举窗口**：找到所有游戏窗口并全选
2. **按键设置**：按键类型选择`W`
3. **开始发送**：所有窗口同步收到`W`指令

#### 场景3：游戏按键无效排查

**问题：** 用方式1发送`A`键没反应

**排查步骤：**
1. 尝试方式2 (PostMessage)
2. 尝试方式3 (keybd_event)
3. 尝试方式4 (SendInput)
4. 尝试方式5 (WM_KEYDOWN直接)
5. 尝试方式21 (DirectInput模拟)

如果都不行，以管理员权限运行，再试方式22 (驱动级模拟)。

### 4.2 软件自动化场景

#### 场景1：批量向聊天窗口发送消息

**需求：** 打开5个聊天窗口，向每个发送相同消息。

**设置步骤：**
1. **枚举窗口**：找到所有聊天窗口并全选
2. **按键设置**：
   - 组合键：勾选`Ctrl`
   - 按键类型：`V`（粘贴）
3. **准备工作**：把要发送的消息复制到剪贴板
4. **开始发送**：所有窗口依次收到`Ctrl+V`

#### 场景2：软件压力测试

**需求：** 向软件快速发送1000次`Enter`键，测试稳定性。

**设置步骤：**
1. **枚举窗口**：找到目标软件窗口
2. **按键设置**：
   - 按键类型：`Enter`
   - 发送间隔：`0.05`秒
   - 重复次数：`1000`
3. **开始发送**：程序快速发送Enter键

### 4.3 系统运维场景

#### 场景1：批量配置服务器

**需求：** 通过远程桌面连接多台服务器，执行相同的命令。

**设置步骤：**
1. **枚举窗口**：找到所有远程桌面窗口
2. **按键设置**：按顺序输入命令（如`ipconfig`）
3. **开始发送**：向所有窗口发送按键

#### 场景2：解决按键无效问题

**问题：** 某个老旧软件按键盘没反应

**解决方案：**
1. 开启调试模式，确认窗口已找到
2. 从方式1开始，依次尝试不同方式
3. 如果方式3或4有效，说明软件读取的是硬件输入
4. 如果都不行，用管理员权限运行

## 5. 高级技巧与优化

### 5.1 动态参数调整

**技巧：** 在发送过程中，可以随时修改按键类型、间隔、重复次数、组合键，修改后立即生效！

**应用：** 编写一个技能循环：
1. 先发送F1
2. 在发送过程中，把按键类型改成F2
3. 等几秒后，改成F3
4. 再改回F1，形成循环

### 5.2 无限循环显示轮数

当重复次数设置为0（无限循环）时，进度条会变成脉冲模式，并显示已完成的发送轮数。您可以随时查看"发送轮数"标签，了解程序运行了多久。

### 5.3 使用虚拟键码发送特殊按键

有些按键在预设列表中没有，可以手动输入虚拟键码：

| 按键 | 虚拟键码 |
|------|----------|
| 空格 | 0x20 |
| 回车 | 0x0D |
| Esc | 0x1B |
| Tab | 0x09 |
| 删除 | 0x2E |

### 5.4 组合键的几种用法

- **单组合键**：Ctrl+C、Alt+F4
- **双组合键**：Ctrl+Alt+Del（需要依次勾选Ctrl和Alt，主键选Del）
- **修饰键单独发送**：直接选择Ctrl作为按键类型

### 5.5 调试模式排查问题

当找不到窗口或发送无效时，开启调试模式：

1. 勾选"调试模式"
2. 点击"刷新窗口列表"
3. 查看日志，找到目标窗口的详细信息
4. 复制正确的关键词重新搜索

## 6. 常见问题解答 (FAQ)

### Q1: 为什么找不到我的游戏窗口？

**A:** 尝试以下步骤：
1. 开启**调试模式**，刷新列表，在日志中搜索游戏窗口
2. 游戏窗口标题可能变化，尝试用"**按程序名**"模式
3. 输入程序加载的**模块名**（如zhengtu.dat），而不是exe文件名
4. 确认游戏窗口是**可见**的（没有被最小化到托盘）

### Q2: 为什么发送按键后目标窗口没反应？

**A:** 按顺序尝试：
1. 切换不同的发送方式（1→2→3→4...）
2. 增大发送间隔（0.5→1.0秒）
3. 以**管理员权限**运行程序
4. 确认勾选的是正确的窗口
5. 如果是游戏，尝试方式21 (DirectInput模拟)

### Q3: "无限循环"模式如何停止？

**A:** 点击红色的 **⏹ 停止发送** 按钮即可。

### Q4: 为什么Fn键无法模拟？

**A:** Fn是硬件级按键，不产生Windows虚拟键码，无法通过软件模拟。这是所有软件的通用限制。

### Q5: 如何以管理员权限运行？

**A:** 右键点击 `KeyForge.exe`，选择"**以管理员身份运行**"。

### Q6: 发送间隔最小可以设置多少？

**A:** 最小0.01秒（10毫秒）。但间隔太小可能导致目标程序反应不过来，建议从0.3秒开始尝试。

### Q7: 为什么有些按键（如F13-F24）没有效果？

**A:** F13-F24是旧式键盘的按键，现代软件和硬件大多不支持。但KeyForge仍然提供支持，供特殊场景使用。

### Q8: 可以同时向多个窗口发送不同的按键吗？

**A:** 目前版本不支持同时发送不同按键。但您可以利用动态参数调整，在发送过程中切换按键类型。

### Q9: 程序会报毒吗？

**A:** 部分杀毒软件可能对DLL注入、驱动级模拟等方式报毒。这是正常现象，因为这类操作确实可能被恶意软件使用。请确保从官方GitHub下载程序。

### Q10: 如何获得技术支持？

**A:** 请通过 GitHub Issues 提交问题，附上：
- 操作系统版本
- 目标程序名称
- 问题描述
- 日志截图

我们不提供私人邮箱支持，以便其他用户也能受益。
