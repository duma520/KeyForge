## 1. 引言：KeyForge是什么？

**KeyForge**（代号“键铸”）是一款功能强大、设计优雅的后台按键测试工具。它就像一个“键盘使者”，能够在您电脑上运行的任何窗口（尤其是那些被最小化或后台运行的游戏、软件）中模拟键盘输入，而不会干扰您当前的鼠标和键盘操作。

无论您是游戏玩家（想测试游戏连招、挂机脚本）、软件测试工程师（想自动化测试软件界面）、还是系统管理员（想批量操作多个窗口），KeyForge都能为您提供一个强大、可靠且易于使用的解决方案。它不仅支持从A到Z的所有字母和数字，还支持F1-F12功能键、方向键、组合键（如Ctrl+C），并提供了**15种不同的后台按键发送方式**，以应对各种复杂场景下的“按键无效”问题。

**核心理念：** 将最专业的后台技术，包装成最简单易用的界面，让每一位用户都能成为自己电脑的“指挥家”。

## 2. 零基础快速入门（5分钟上手）

别担心，即使您没有任何技术背景，也能在几分钟内让KeyForge工作起来。

### 2.1 第一步：启动与界面初识

下载并运行 `KeyForge.exe`（或通过 `1.bat` 运行），您会看到一个色彩温和的界面。整个界面分为上下两部分：

-   **上半部分：** “窗口枚举设置” 和 “窗口列表”，用于寻找和选择目标。
-   **下半部分：** “按键设置”、“执行进度” 和 “执行日志”，用于配置和控制。

### 2.2 第二步：找到你的目标窗口

1.  **确定你的目标**：假设你想向一个名为“记事本”的窗口发送按键。
2.  **选择枚举方式**：在“窗口枚举设置”区域，选择“**按窗口标题**”。
3.  **输入关键词**：在“窗口标题关键词”框中，输入“**记事本**”。
4.  **刷新列表**：点击 **🔍 刷新窗口列表** 按钮。稍等片刻，下方列表就会显示所有标题包含“记事本”的窗口。
5.  **勾选目标**：在窗口列表中，找到您想操作的记事本窗口，勾选它前面的复选框。

### 2.3 第三步：选择要发送的按键

1.  **设置按键**：在下半部分的“按键设置”区域，点击“按键类型”下拉框，选择“**A**”。
2.  **设置发送方式**：保持“后台按键方式”为默认的“**1. SendMessage**”。这是最基础、最通用的方式。
3.  **设置重复次数**：保持“重复次数”为“**1**”。这意味着只发送一次。

### 2.4 第四步：开始发送！

1.  点击绿色的 **▶ 开始发送按键** 按钮。
2.  观察您的“记事本”窗口，它应该会接收一个“A”字符。
3.  查看下方的“执行日志”区域，您会看到类似 `[时间] SendMessage: 已发送 A 键` 的日志记录。
4.  发送完成后，进度条会显示100%，状态栏会变为“发送完成”。

恭喜您！您已经成功完成了第一次后台按键发送。

## 3. 核心功能详解（从入门到精通）

### 3.1 窗口枚举设置（如何精准“瞄准”目标）

这部分是KeyForge的“眼睛”，决定了它能“看”到哪些窗口。

#### 3.1.1 按窗口标题
**这是什么？** 程序会搜索所有可见窗口的标题栏文字，只要包含您输入的关键词，就会被找到。
**适用场景：** 目标窗口的标题很明确且独一无二，例如“记事本”、“Word - 文档1”。
**举例：**
-   输入“**征途**”：会找到所有标题中含有“征途”二字的窗口，如“征途2”、“征途怀旧版”。
-   输入“**我的世界**”：会找到Minecraft的游戏窗口。

#### 3.1.2 按程序名
**这是什么？** 程序会搜索与您输入的关键词相匹配的**进程名**（即 `.exe` 文件名），甚至是它加载的**模块名**（如 `.dll` 或 `.dat` 文件）。这对于那些标题会变化但程序名固定的游戏非常有效。
**适用场景：** 窗口标题会动态变化（如显示玩家名、当前时间），但程序文件是固定的。
**举例：**
-   **普通匹配**：输入“**notepad.exe**”，可以匹配到记事本程序的所有窗口。
-   **模块匹配（针对游戏）**：输入“**zhengtu.dat**”。即使游戏主程序名是 `client.exe`，但进程加载了 `zhengtu.dat` 模块，KeyForge依然能精准找到。这是解决一些网游（如征途系列）后台化问题的关键。

#### 3.1.3 两者兼顾
**这是什么？** 同时满足“标题包含关键词”和“程序名包含关键词”两个条件。
**适用场景：** 当您运行了多个同程序名的窗口，需要根据标题进一步筛选时。例如，开了两个“记事本”，一个编辑“文件A.txt”，一个编辑“文件B.txt”，您可以通过标题“文件A”和程序名“notepad.exe”来精确控制。

#### 3.1.4 调试模式：打开“透视眼”
**这是什么？** 勾选“调试模式”后，点击“刷新窗口列表”，程序会将所有可见窗口的详细信息输出到“执行日志”区域。
**作用：** 这是解决问题的利器！当您找不到目标窗口时，开启调试模式，就能看到系统里所有窗口的标题、类名、PID、进程名和加载的模块名。您可以从中找出您想要的窗口，并复制其**窗口标题**或**程序名关键词**，然后关闭调试模式，用正确的关键词进行搜索。

### 3.2 按键设置（15种发送方式与组合键）

这是KeyForge的“嘴巴”，决定了它如何“说”出按键。

#### 3.2.1 按键类型：支持所有你能想到的键
KeyForge内置了极其丰富的按键库，包括：
-   **字母：** A-Z
-   **数字：** 0-9
-   **功能键：** F1 - F12
-   **方向键：** Up, Down, Left, Right
-   **控制键：** Space, Enter, Esc, Tab, Backspace, Delete, Insert, Home, End, PageUp, PageDown
-   **修饰键：** Ctrl, Alt, Shift, Win (Windows徽标键)
-   **小键盘：** Num0-9, NumMult, NumPlus...
-   **符号键：** `, -, =, [, ], \, ;, ', ,, ., /
-   **其他：** CapsLock, PrintScreen, ScrollLock, Pause, Menu

**高级用法：** 您甚至可以手动输入**虚拟键码（VK Code）**！例如，输入 `0x41` 代表字母 `A`，输入 `0x20` 代表空格。这为需要发送特定键位或自定义按键的用户提供了最大灵活性。

#### 3.2.2 组合键：一键触发复杂操作
**这是什么？** 可以同时按下多个键，模拟“Ctrl+C”、“Alt+F4”这样的组合操作。
**如何使用：** 在“按键设置”区域，您会看到一排复选框：`Ctrl`、`Alt`、`Shift`、`Win`。勾选它们，然后选择一个主按键（如 `C`）。程序发送时，会先按下所有选中的组合键，再按下主键，然后依次释放。
**适用场景：**
-   **软件操作**：`Ctrl+S` 保存文件，`Ctrl+A` 全选。
-   **游戏技能**：`Ctrl+1` 使用技能。
-   **系统命令**：`Alt+Tab` 切换窗口，`Win+R` 打开运行。

#### 3.2.3 发送间隔与重复次数
-   **发送间隔(秒)**：每两次按键发送之间的等待时间。最小值0.01秒，最大值10秒。如果程序反应慢，可以适当调高此值。
-   **重复次数**：
    -   **大于0的数字**：表示发送指定次数。
    -   **0**：**无限循环**模式！程序会持续不断地向目标窗口发送按键，直到您点击 **⏹ 停止发送** 按钮为止。非常适合挂机或压力测试场景。

#### 3.2.4 15种后台按键方式深度解析

这是KeyForge最核心的专业功能。不同的游戏和软件对键盘消息的处理方式不同，有的只接受Windows消息，有的则依赖更底层的硬件输入。KeyForge为您准备了15种“武器”，让您总能找到攻破对方“防线”的那一种。

| 序号 | 方式名称 | 通俗解释 | 原理 | 适用场景与优缺点 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | **SendMessage** | **“快递员直接敲门”** | 直接将按键消息放入目标窗口的消息队列，并**等待**其处理完成。 | 最稳定、最基础的方式。如果游戏或软件本身支持消息处理，这种方式首选。**优点：** 稳定，易实现。**缺点：** 可能被某些高级输入钩子过滤。 |
| **2** | **PostMessage** | **“快递员投递后离开”** | 将按键消息放入目标窗口的消息队列，**不等待**处理结果。 | 比`SendMessage`更快，但成功率可能稍低。常用于快速连续操作。 |
| **3** | **keybd_event** | **“全局扩音器”** | 模拟全局的键盘硬件事件，效果如同您真实按下键盘，会影响所有窗口。 | 当后台方式无效时的备选方案。**注意：** 此方式会影响您当前正在使用的窗口，可能会造成干扰。 |
| **4** | **SendInput** | **“增强版扩音器”** | 比`keybd_event`更先进、更安全的全局输入模拟，能模拟更复杂的输入序列。 | Windows官方推荐的输入模拟方式，兼容性好，是`keybd_event`的升级版。 |
| **5** | **WM_KEYDOWN直接** | **“特殊格式的快递”** | 发送带有完整`lParam`参数的按键消息，模拟了键盘扫描码、扩展键等信息。 | 某些游戏会校验`lParam`参数，用此方式可能“蒙混过关”。 |
| **6** | **WM_CHAR** | **“发送最终的字符”** | 直接发送字符消息，而不是按键的按下和抬起。 | 适用于需要输入文本的场景，但不适用于游戏技能释放（因为游戏通常监听的是按键而非最终字符）。 |
| **7** | **WM_IME_CHAR** | **“输入法专用字符”** | 模拟输入法产生的字符消息。 | 特定场景下需要输入中文或特殊符号时使用。 |
| **8** | **WM_SYSKEYDOWN** | **“系统快捷键”** | 模拟系统级按键（如`Alt+F4`、`Alt+Tab`）。 | 专门用于发送组合键，尤其是涉及`Alt`键的系统级组合。 |
| **9** | **PostMessage完整lParam** | **“更细节的快递”** | 结合了2和5的优点，使用`PostMessage`异步发送，但携带完整的`lParam`参数。 | 平衡了速度和参数真实性，是后两种方式的结合体。 |
| **10** | **SetWindowsHookEx钩子注入** | **“卧底”** | 通过Windows钩子，将一段代码“注入”到目标窗口所在的线程，再从内部模拟按键。 | 非常强大，但实现复杂，成功率取决于目标程序的防注入机制。 |
| **11** | **硬件输入模拟** | **“硬件级模拟”** | 直接调用`keybd_event`等底层API的变体。 | 与方式3、4类似，属于全局模拟的另一种实现。 |
| **12** | **DirectInput模拟** | **“游戏手柄模拟”** | 模拟DirectInput设备的输入，这是许多游戏直接读取的输入方式。 | 专为使用DirectInput API的游戏设计，是解决游戏后台按键问题的“利器”。 |
| **13** | **驱动级模拟** | **“操作系统底层”** | 尝试通过操作键盘控制器端口（`0x60`）来模拟硬件信号。 | 理论上的“终极”模拟方式，**需要管理员权限**且成功率受系统环境影响很大。 |
| **14** | **VMware虚拟HID** | **“虚拟机专属”** | 尝试通过VMware的后门接口发送HID（人机交互设备）信号。 | 专门为在VMware虚拟机中运行的目标程序设计。 |
| **15** | **DLL注入** | **“完全接管”** | 将一个动态链接库（DLL）注入到目标进程，然后在其内部调用`keybd_event`等函数。 | 最高权限的模拟方式，几乎可以应对任何防护，但也是最危险、最容易被杀毒软件拦截的方式。 |

**实战建议：** 不知道选哪个？先从**1 (SendMessage)** 和 **2 (PostMessage)** 开始。如果无效，尝试**3 (keybd_event)** 或**4 (SendInput)**。如果还不行，说明目标程序可能使用了DirectInput或对消息有特殊校验，请尝试**5 (带完整lParam)** 或 **12 (DirectInput模拟)**。如果所有方式都无效，可考虑以**管理员权限**运行程序，再尝试方式13-15。

## 4. 高级应用与场景举例

### 4.1 游戏辅助测试

#### 场景1：自动连招（例如《征途》）
-   **目标：** 在《征途》游戏中，模拟按下技能键 `F1`、`F2`、`F3`，实现一键连招。
-   **设置步骤：**
    1.  **枚举窗口：** 使用“按程序名”模式，输入关键词“**zhengtu.dat**”，点击刷新，勾选所有匹配的游戏窗口。
    2.  **按键设置：** 将“按键类型”设置为 **F1**。
    3.  **循环模式：** 将“重复次数”设置为 **0**（无限循环）。
    4.  **开始发送：** 点击“开始发送”。此时游戏窗口会不断收到`F1`。为了发送`F1`、`F2`、`F3`的循环，您可以借助`KeyForge`的**动态参数调整**功能：在发送过程中，依次修改“按键类型”为F2、F3，程序会实时生效，完成一套连招循环。

#### 场景2：无限循环挂机
-   **目标：** 在一个挂机游戏中，每隔3秒按下“空格键”以保持角色状态。
-   **设置步骤：**
    1.  **枚举窗口**：找到游戏窗口。
    2.  **按键设置：** 选择按键类型为 **Space**，发送间隔设为 **3.00** 秒，重复次数设为 **0**。
    3.  **开始发送：** 点击开始，即可实现无限循环挂机。

### 4.2 软件自动化测试

#### 场景1：跨窗口批量操作（如聊天工具）
-   **目标：** 打开多个聊天窗口，向每个窗口依次发送一条消息，然后关闭。
-   **设置步骤：**
    1.  **枚举窗口：** 用“按窗口标题”模式，输入关键词，找到所有聊天窗口并全选。
    2.  **按键设置：** 在“按键类型”中手动输入您的消息内容（需要先复制到剪贴板，然后通过`Ctrl+V`发送）。可以设置组合键`Ctrl`+`V`来模拟粘贴操作。
    3.  **开始发送：** 程序会依次向每个窗口发送`Ctrl+V`。然后再发送`Enter`键发送消息。

#### 场景2：模拟复杂组合键操作（如Ctrl+S保存）
-   **目标：** 自动保存多个打开的文本文档。
-   **设置步骤：**
    1.  **枚举窗口：** 找到所有打开的“记事本”窗口。
    2.  **按键设置：** 勾选组合键 **Ctrl**，主按键选择 **S**。
    3.  **开始发送：** 所有记事本窗口都会同时（或依次）收到`Ctrl+S`指令并自动保存。

### 4.3 系统运维与调试

#### 场景1：解决“按键无响应”问题
-   **目标：** 向一个特殊的软件（如老旧的控制台程序）发送按键，但SendMessage无效。
-   **排查步骤：**
    1.  **启用调试模式**，刷新窗口，确认已找到正确的窗口。
    2.  **从方式1开始，依次尝试方式2、3、4...** 观察日志和软件反应。
    3.  如果方式3或4有效，说明该程序可能不处理窗口消息，而是直接读取硬件输入。
    4.  如果仍然无效，尝试**以管理员权限**运行KeyForge，再尝试方式13、15等。

#### 场景2：多窗口同步操作
-   **目标：** 操作一个多开的游戏，实现同步跑动或施法。
-   **设置步骤：**
    1.  **枚举窗口：** 通过“按程序名”或“按窗口标题”找到所有游戏窗口，并**全选**。
    2.  **按键设置：** 选择`W`键（前进）或`D`键（攻击）。
    3.  **开始发送：** 所有窗口会同步收到按键指令，实现多窗口同步操作。

## 5. 常见问题解答 (FAQ)

**Q1: 为什么找不到我的游戏窗口？**
**A:** 请尝试以下步骤：
1.  **开启调试模式**，点击刷新，在日志中搜索游戏窗口的特征（如标题、进程名、模块名）。
2.  如果标题关键词不准确，请复制日志中显示的完整窗口标题。
3.  如果游戏进程是 `game.exe`，但游戏加载了 `mygame.dat`，请使用“按程序名”模式，并输入 `mygame.dat` 作为关键词。

**Q2: 为什么发送按键后目标窗口没反应？**
**A:** 原因可能有多种：
1.  **方式不对：** 当前选择的“后台按键方式”不被该窗口识别。请参考本文第3.2.4节的深度解析，**尝试切换不同的发送方式**。
2.  **权限不足：** 某些游戏或软件需要管理员权限才能接收后台消息。请**以管理员身份运行**KeyForge。
3.  **目标错误：** 窗口列表中的窗口已失效（如游戏崩溃、关闭）。请重新刷新列表。
4.  **间隔太快：** 某些软件处理速度慢。请尝试**增大“发送间隔”**。

**Q3: “无限循环”模式如何停止？**
**A:** 非常简单，点击红色的 **⏹ 停止发送** 按钮即可。

**Q4: 为什么有些按键（如Fn）无法模拟？**
**A:** `Fn`键是硬件级的按键，它通常不产生标准的Windows虚拟键码，而是直接由硬件和驱动程序处理，因此绝大多数软件都无法模拟它。

**Q5: 如何以管理员权限运行？**
**A:** 右键点击 `KeyForge.exe` 文件，选择“**以管理员身份运行**”。
